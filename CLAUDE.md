Anchors: leading `/` = project root, `//` = fs-absolute, bare name = any depth. Rules hot-reload → after each edit Read-test one path that MUST block + one that MUST stay readable (catches over- and under-deny). A `Read()` deny also gates `Bash`, leakily: a cmd naming a matching path as an arg is blocked (`grep`/`stat`/`jq`; `find` too but RTK rewrites simple `find`→`rtk find`, so only piped `find` blocks) while some slip through (`ls`/`wc`/`echo`); the check is static on command text → to read a denied file deliberately, use runtime indirection (a script/module that `open()`s it, path never in the command).
