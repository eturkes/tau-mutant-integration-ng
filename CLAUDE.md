- Deny-`Read()` globs (`permissions.deny`) = gitignore-style, silently over-/under-matching → verify each by Read-test (eyeballing misses it). `dir/*/**` over-denies: `*` = one segment incl. a bare filename + `/**` = zero+ segments → blocks `dir/child` as well as `dir/sub/deep`; prefer `**/*.ext` (binary/dump trees) + exact-path rules. Anchors: leading `/` = project root, `//` = fs-absolute, bare name = any depth. Rules hot-reload → after each edit Read-test one path that MUST block + one that MUST stay readable (catches over- and under-deny). A `Read()` deny also gates `Bash`, leakily: a cmd naming a matching path as an arg is blocked (`grep`/`stat`/`jq`; `find` too but RTK rewrites simple `find`→`rtk find`, so only piped `find` blocks) while some slip through (`ls`/`wc`/`echo`, and any cmd naming only a parent dir — a root-wide `grep -r` dumps denied content); the check is static on command text → to read a denied file deliberately, name just a parent dir (`grep -r`) or use runtime indirection (a script/module that `open()`s it, path kept off the command line).
